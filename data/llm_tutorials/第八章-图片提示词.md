## 插图6：三个问题解决

**位置**：8.5 为什么需要MCP？

**图片描述**：
对比图，左边"以前"（各自为政，混乱），右边"现在"（MCP统一，有序），展示三个问题的解决：
1. 统一接口（不用每个数据源单独对接）

**画面风格**：
- 左右对比

- 红vs绿
- 解决方案主题

**提示词（英文）**：
```
Before vs After comparison, left: each data source needs separate connection (chaos, red), right: MCP unified standard (orderly, green), showing three solutions: unified interface, security with permission control, flexibility by changing servers
```

**提示词（中文）**：
```
以前vs现在对比，左边：每个数据源需要单独连接(混乱，红色)，右边：MCP统一标准(有序，绿色)，展示三个解决方案：统一接口、安全性(权限控制)、灵活性(更换服务器)
```

## 图片生成建议

### 第8章特点

**主题**：MCP连接大模型与外部世界

**关键概念**：
- 知识局限
- 统一接口
- 三层架构
- 资源/提示/工具
- 安全性和灵活性

**视觉策略**：
- 使用桥梁、连接的隐喻
- USB接口比喻
- 对比图（孤立vs连接，以前vs现在）
- 流程图展示架构

**颜色建议**：
- 连接/桥梁：蓝色、绿色
- 局限/问题：橙色、红色、灰色
- MCP统一标准：蓝色、紫色
- 应用场景：明亮色彩

### 推荐工具

1. **Midjourney**：场景化插图
2. **DALL-E 3**：概念图和架构图
3. **Canva**：后期编辑和文字

---

**提示**：第8章重点是MCP的"连接"作用，图片需要突出"桥梁"和"统一接口"的概念，展示MCP如何打破大模型的局限。
